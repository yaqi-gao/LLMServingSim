## Key Design Decisions

### 1. head_dim Override
The most critical change. GPT-OSS-20B has `hidden_size=2880` and `num_attention_heads=64`, giving a naive `head_dim=45`. But the actual model uses `head_dim=64` (total Q projection size = 64*64 = 4096 != 2880). The SGLang source confirms this via `getattr(config, "head_dim", ...)`.

The fix uses a fallback pattern: `config.get('head_dim', n_embd // n_head)` — existing models without explicit `head_dim` are unaffected.

### 2. MoE Structure Reuse
GPT-OSS-20B uses the same w1/w2/w3 SwiGLU expert structure as Mixtral. The simulator's existing MoE pipeline (gate → ALLTOALL → per-expert w1/w2/w3 → ALLTOALL) applies directly. No changes to `trace_generator.py` MoE logic or `gate_function.py` were needed.

### 3. Profiler Model Pattern
`llm_profile/models/gpt_oss.py` follows the established pattern from `mixtral.py`:
- Single expert instance for profiling (scaled by `num_local_experts // tp` in the profiler main)
- Timer instrumentation on all relevant ops
- Attention uses explicit `head_dim` from config

## What's NOT Implemented (Future Work)

1. **Sliding window attention**: All layers use full causal attention in the simulator. GPT-OSS-20B alternates sliding-window (128 tokens) and full attention per layer. Full attention is a conservative (slightly pessimistic) approximation.

2. **YaRN RoPE scaling**: The simulator doesn't model RoPE computation cost differences. Negligible impact on overall latency.

3. **MXFP4 quantization**: GPT-OSS-20B has an MXFP4-quantized variant. The simulator uses uniform `--fp` precision. Mixed-precision expert quantization is not supported.

4. **GPT-OSS-120B**: Same architecture with 36 layers and 128 experts. Can be added by creating `model_config/openai/gpt-oss-120b.json` and a corresponding cluster config.

5. **Profiling data**: Synthetic profiling data was generated by scaling Mixtral-8x7B H100 measurements using FLOP/memory ratios (see `llm_profile/generate_gpt_oss_profile.py`). For production-quality results, re-profile on actual H100 hardware.

## Profiling Data

Synthetic profiling data was generated by scaling Mixtral-8x7B H100 measurements:
```bash
cd llm_profile && python3 generate_gpt_oss_profile.py
```

Generated files:
- `llm_profile/perf_models/H100/openai/gpt-oss-20b/tp1/layers.csv` (30,720 rows)
- `llm_profile/perf_models/H100/openai/gpt-oss-20b/tp2/layers.csv` (30,720 rows)
- `llm_profile/perf_models/H100/openai/gpt-oss-20b/tp4/layers.csv` (30,720 rows)

Key scale factors (gpt-oss / mixtral):
- Expert w1/w2/w3: 0.14x (2880 vs 14336 intermediate)
- Gate: 2.8x (32 vs 8 experts)
- Embedding/lm_head: 4.4x (201K vs 32K vocab)
- Q/O proj: 0.7x (similar Q size, smaller hidden)

For production-quality results, re-profile on actual H100 GPU:
```bash
cd llm_profile
python -m profiler.layers.main --model openai/gpt-oss-20b --hardware H100 --tp-size 1 --max-len 2048
```

## How to Run

### Run simulation (requires ASTRA-sim build via ./compile.sh)
```bash
python main.py \
    --cluster-config 'cluster_config/single_node_gpt_oss_moe_single_instance.json' \
    --fp 16 --block-size 16 \
    --dataset 'dataset/example_trace.jsonl' \
    --output 'output/example_gpt_oss_moe_run.csv' \
    --num-req 10
```
